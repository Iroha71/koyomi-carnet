# Androidコードレビュー

このスキルは、こよみのカルネ（Kotlin / Jetpack Compose Androidアプリ）の実装を、3つの専門サブエージェントに並列でレビューさせ、結果を統合して提示するためのものです。

## 手順

1. **レビュー対象の特定**
   - ユーザーが特定のファイル・機能を指定していればそれを対象にする。
   - 指定がなければ、まず `git status` / `git diff` で作業中の変更を確認し、変更のあるファイルを対象にする。
   - 差分もローカル変更もない場合は、ユーザーに対象（ブランチ名・PR・ファイルパスなど）を確認する。

2. **3つの専門サブエージェントを並列で呼び出す**
   - Agent ツールを **1回のメッセージで3つ同時に** 呼び出すこと（順番に呼ばない）。
   - それぞれ `subagent_type` に以下を指定する。
     - `android-performance-reviewer` — パフォーマンス観点
     - `android-security-reviewer` — セキュリティ観点
     - `kotlin-code-reviewer` — Kotlin構文・コード構成観点
   - 各エージェントへのプロンプトには、以下を必ず含める。
     - レビュー対象のファイルパス一覧、または対象の差分（`git diff` の内容）
     - 「このリポジトリの該当ファイルをReadで実際に読んだ上でレビューすること」という指示
     - 出力は各エージェント自身のフォーマット（`[重大|警告|提案]` の箇条書き、日本語）に従うこと

3. **結果の統合**
   - 3エージェントの回答を待ち、以下の方針でレビュー結果をまとめる。
     - 観点ごと（パフォーマンス／セキュリティ／Kotlin構文・構成）に見出しを分けて列挙する。
     - 複数のエージェントが同じ箇所を指摘している場合は重複を整理し、まとめて記載する。
     - 重大度（重大 > 警告 > 提案）順に並べる。
     - 指摘が0件の観点は「重大な問題は見つかりませんでした」と明記する。
   - 最終的なレビュー結果は必ず日本語で出力する（[.github/copilot-instructions.md](../../../.github/copilot-instructions.md) のレビュー言語規約に準拠）。

4. **まとめ**
   - 全体の総評（マージ可否の目安、優先して直すべき指摘）を最後に1〜2文で添える。
   - 指摘に対する自動修正は行わず、まずレビュー結果の提示に留める。修正が必要な場合はユーザーの指示を待つ。

## 注意事項

- レビュー観点の3エージェントはいずれも読み取り専用（Read/Grep/Glob）で、コードの変更は行わない。
- サブエージェントには実際にファイルを読ませ、推測や一般論だけで指摘させないこと。
- アーキテクチャ規約は [.claude/rules/ui.md](../../rules/ui.md)、開発規約は [.claude/rules/development.md](../../rules/development.md) を参照。
